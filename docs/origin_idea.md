全网搜集skills、提示词形成可沉淀的知识库，使用 milvus mcp 存储。构建一个提示词语义检索生成的网站。

产品工作流：
1用户输入query，将query输入中控（LLM+提示词），对query进行处理（重写、润色或者拆分）使其更利于检索，返回一个列表给下方流程（即使最后只有一个关键词也是以列表下发）。
FOR sub_task IN tasks:
  2遍历上面列表中的全部关键词或短语，到 milvus 云端向量库检索。如果检索到相关性非常高的结果，则下发给下游。如果没有相关性达标的内容，或者内容数量不够，则联网检索，将检索到的结果和来源存储到向量库并且下发。这里从向量库检索或者网上搜索下发的内容，除了包含原本的内容，还必须包含内容来源。
  3将检索到的内容，与列表中拆解的子关键词/任务，输入给agent（LLM+提示词）进行整理﻿
END
4将整理得到的所有内容+子关键词/任务名称，与用户最初的query一起输入agent（LLM+提示词）得到最终结果

产品技术模块：
信息采集模块：exa联网搜索、serper联网搜索、github仓库搜索阅读获取工具、arxiv论文搜索阅读工具﻿
Agent模块（元提示词）：query拆解与任务规划提示词、子任务的信息整理与筛选融合提示词、信息汇总生成最终答案的提示词
Milvus MCP：用于连接milvus向量数据库

产品技术栈：
前端：vite、react、Tailwindcss、shadcn ui
后端和算法：FastAPI、python

产品交互过程：
在整个过程中，用户应该能实时看到每一步的变化，并可以从任何一个步骤修改，修改后从该步骤重新运行。
在第一步，将用户query拆分成子任务列表后，等待用户确认后再执行。且支持用户修改子任务列表。
在第二步，以折叠卡片的形态展示搜索到的结果，并标注数据来源，若有链接则给出使得可跳转。若需要联网搜索，则展示联网搜索过程。最终结果展示同上。
在第三步，在检索到的内容卡片中，以子卡片的形式展示生成的汇总/整理的最终结果。
在第四步，展示生成的最终结果。
